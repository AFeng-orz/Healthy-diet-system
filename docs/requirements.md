# 需求文档

## 1. 项目概述

**项目名称**：健康生活饮食规划系统

**项目目标**：构建一个面向减脂健身人群的健康饮食管理系统，通过身体档案、健康画像问卷、营养计算、食物库和饮食计划生成，帮助用户制定并追踪合理的饮食计划。

**目标用户**：希望减脂、改善饮食结构、建立健康生活习惯的普通用户和健身入门人群。

**边界声明**：本系统只提供生活方式建议，不提供医疗诊断或治疗建议。

## 2. 核心功能规划

### 2.1 用户体系

- 用户注册、登录、退出。
- 每个用户拥有独立的身体档案、问卷结果、饮食计划和记录。

### 2.2 健康档案

- 记录性别、年龄、身高、体重、目标体重、运动频率、饮食目标、忌口、过敏和偏好。
- 自动计算 BMI、BMR、TDEE、推荐热量和三大营养素目标。

### 2.3 健康画像问卷

- 通过问卷评估饮食结构、运动习惯、作息压力、饮水习惯、进食习惯。
- 输出饮食不均衡型、运动不足型、作息压力型、饮水不足型、高热量偏好型等标签。

### 2.4 食物营养库

- 维护常见食物的热量、蛋白质、脂肪、碳水和分类信息。
- 支持分页、筛选、新增、编辑、删除。

### 2.5 饮食计划生成

- 根据用户档案、目标热量、营养素目标、忌口偏好和健康画像生成一日三餐计划。
- 每餐包含食物、推荐克数、热量、营养素和推荐原因。
- Phase 7 第一版采用规则引擎生成：
  - 早餐约占推荐热量 25%。
  - 午餐约占推荐热量 40%。
  - 晚餐约占推荐热量 35%。
  - 每餐优先组合主食、蛋白质、蔬菜或水果。
  - 食物选择需要同时满足分类和基础营养特征，例如主食应具备一定碳水和热量，蛋白质食物应具备一定蛋白质含量。
  - 推荐克数需要落在常见份量范围内，避免因选择低热量食物导致全天总热量明显偏低。
  - 自动避开健康档案中记录的忌口和过敏关键词。
  - 忌口支持自然语言短句中的常见写法，例如“不吃鸡蛋和水果”会识别为避开鸡蛋相关食物和水果分类。
  - 当某类食物被忌口过滤后，允许使用更适合餐次的备选分类补位，例如早餐水果被过滤时可尝试乳制品。
  - 推荐说明结合饮食目标和最近一次健康画像。
- 当前计划生成仅作为生活方式参考，不作为医疗处方或严格营养配餐。

### 2.6 饮食记录

- 用户记录每日摄入食物、克数和餐次。
- 系统自动计算每日总热量和营养素摄入。
- Phase 8 第一版支持：
  - 选择日期、餐次、食物和摄入克数。
  - 根据食物库每 100g 营养数据自动换算本次摄入的热量、蛋白质、脂肪和碳水。
  - 查看当天热量和三大营养素总摄入。
  - 与健康档案中的推荐热量和三大营养素目标做完成度对比。
  - 删除误添加的饮食记录。

### 2.7 健康报告

- 展示今日达成率、周/月热量趋势、体重趋势、营养结构和计划执行率。
- Phase 9 第一版支持：
  - 查看今日热量、蛋白质、脂肪和碳水完成率。
  - 查看最近 7 天热量摄入趋势。
  - 统计最近 7 天有记录的天数和日均热量。
  - 根据今日热量完成度生成简短生活方式建议。
