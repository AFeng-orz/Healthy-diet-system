# 项目工作说明

## 1. 迭代原则

本项目必须严格按阶段小步推进。

- 不一次性完成所有功能。
- 每次只完成一个可验证闭环。
- 每个阶段完成后更新 docs/ 与 logs/。
- 每个阶段运行对应验证命令。
- 若阶段仅涉及文档，需要在日志中明确说明未运行构建命令。
- 当前阶段之外的功能只记录为待办，不提前实现。

## 2. 阶段顺序

| 阶段 | 名称 | 目标 |
|---|---|---|
| Phase 0 | 文档体系与项目规划 | 建立项目目录、文档、日志和协作规则 |
| Phase 1 | 后端工程骨架 | 搭建 Spring Boot 基础结构，可测试可启动 |
| Phase 2 | 前端工程骨架 | 搭建 Vue 基础结构，可构建可导航 |
| Phase 3 | 用户注册登录 | 完成用户体系和数据隔离 |
| Phase 4 | 健康档案与营养计算 | 完成身体档案和核心指标计算 |
| Phase 5 | 健康画像问卷 | 完成非医疗诊断型生活方式评估 |
| Phase 6 | 食物营养库 | 完成食物数据维护和查询 |
| Phase 7 | 饮食计划生成 | 完成规则引擎生成一日三餐 |
| Phase 8 | 每日饮食记录 | 完成摄入记录与目标对比 |
| Phase 9 | 仪表盘与报告 | 完成趋势和可视化分析 |
| Phase 10 | 简历增强与收尾 | 完成测试、README、截图和简历描述 |

## 3. 日志规则

每天开发结束必须创建或更新 `logs/YYYY-MM-DD.md`，记录已完成事项、问题与解决、验收结果和待办事项。

## 4. 验收命令

后端阶段默认运行：

```powershell
cd backend
mvn test
```

前端阶段默认运行：

```powershell
cd frontend
npm run build
```

文档阶段不运行构建命令，但日志必须说明原因。

## 5. 健康建议边界

系统只提供健康生活和饮食管理建议，不进行疾病诊断、治疗建议或医疗处方推荐。
