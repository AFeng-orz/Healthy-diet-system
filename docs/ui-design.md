# UI 设计规范

## 1. 视觉方向

整体参考 Apple 官网 / Apple Health：简洁、高级、自然、轻盈。

关键词：浅色背景、留白充足、圆角卡片、柔和阴影、健康绿色点缀、干净字体层级、数据卡片化、图表柔和。

## 2. 色彩规范

| 用途 | 建议色值 | 说明 |
|---|---|---|
| 页面背景 | #F7F8F6 | 温和浅灰白 |
| 主色 | #34C759 | 健康绿色，接近 Apple Health 气质 |
| 主文字 | #1D1D1F | 高级黑 |
| 次级文字 | #6E6E73 | 柔和灰 |
| 卡片背景 | #FFFFFF | 白色卡片 |
| 边框 | #E5E5EA | 轻边框 |
| 警示色 | #FF9F0A | 温和橙色 |
| 错误色 | #FF3B30 | 系统红 |

## 3. 组件风格

- 卡片圆角建议 18-24px。
- 页面大块区域保持宽松间距。
- 不使用厚重边框和强烈阴影。
- 表格优先简洁，避免密集后台系统风格。
- 首页以健康指标卡片、环形进度和柔和趋势图为主。
- 表单控件保持大尺寸、圆角、清晰标签。
- 桌面端主内容最大宽度为 1360px，保证首页、列表和记录页面有更舒展的横向空间。
- 页面加载遮罩使用半透明浅色和健康绿色加载图标，避免页面切换时出现默认方角白块。
- 除首页外，顶部导航在下滑浏览内容时自动隐藏，上滑或回到顶部时显示，减少内容阅读遮挡。
- 登录页面向普通用户，避免直接展示 JWT、Token、BCrypt 等技术术语。

Phase 0 只定义视觉规范，不实现页面。

## 6. Phase 7.2 饮食计划页面优化

- 饮食计划页面以“总览 + 三餐卡片”为主，不在每个食物项中重复长段推荐说明。
- 食物项保留名称、分类、推荐克数、热量和三大营养素，信息表达尽量简洁。
- 营养数据使用轻量胶囊标签展示，减少大段文字造成的页面臃肿。
- 详细推荐逻辑保留在后端数据中，前端当前优先展示用户最需要快速扫读的信息。

## 7. 通用危险操作弹窗

- 删除类确认弹窗统一使用 `hd-danger-confirm`。
- 弹窗使用白色大圆角、柔和阴影、浅红提示区和圆角按钮。
- 取消按钮使用浅灰背景，确认删除按钮使用柔和红色渐变。
- 食物库和饮食记录的删除确认保持同一视觉风格，避免不同页面出现割裂感。

## 8. 首页仪表盘

- 首页不再展示开发阶段说明，优先展示真实健康数据摘要。
- 首屏展示今日热量完成率、今日摘要和最近建议。
- 下方展示热量、蛋白质、脂肪、碳水四个目标完成卡片。
- 最近 7 天热量趋势使用轻量横向条形图，不引入复杂图表依赖。
- 常用入口使用简洁列表卡片，帮助用户快速进入健康档案、饮食记录、饮食计划和健康报告。
- 常用入口图片资源放在 `frontend/src/assets/ui/`，在 Vue 文件中通过 `import` 引入，不直接把相对路径字符串写进 `src`。
