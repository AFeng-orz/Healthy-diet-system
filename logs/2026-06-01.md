## 今日问题与解决

| 问题 | 原因 | 解决 |
|---|---|---|
| Windows 终端执行 `npm run dev` 可能被策略拦截 | PowerShell 执行策略限制 `npm.ps1` | 改用 `npm.cmd run dev`、`npm.cmd run build` |
| 登录后右上角显示固定“用户” | 前端没有正确读取当前登录用户信息 | 使用本地登录态和 `/api/auth/me` 同步当前用户 |
| 退出登录弹窗样式普通 | Element Plus 默认弹窗与 Apple Health 风格不一致 | 调整按钮、颜色和弹窗观感，靠近浅色健康主题 |
| 健康问卷页面出现“系统异常” | 前后端接口或后端服务状态不一致 | 检查问卷接口、后端服务和登录态后恢复正常 |
| 生活方式风险分缺少对比 | 页面只显示分数，没有说明最高分或风险含义 | 增加分值范围说明，让用户能判断高低 |
| Windows/PowerShell 中文输出乱码 | 控制台编码和 UTF-8 文件显示不稳定 | 后续文档与日志统一使用 UTF-8，并尽量避免用 PowerShell 直接判断中文文件是否损坏 |
