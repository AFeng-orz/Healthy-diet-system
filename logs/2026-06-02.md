## 今日问题与解决

| 问题 | 原因 | 解决 |
|---|---|---|
| 食物库页面没有初始化数据 | `schema.sql` 已有数据，但后端启动时未自动执行资源目录 SQL | 配置 `spring.sql.init.mode=always` 和 `schema-locations=classpath:schema.sql` |
| 食物库出现新旧重复数据 | 旧表可能没有唯一索引，`INSERT IGNORE` 无法阻止重复插入 | 改为临时表 + `WHERE NOT EXISTS` 补齐，并增加同名去重 SQL |
| 部分高糖/高脂标签不准确 | 早期规则把“相对脂肪更高”和“高脂”混用，也把完整水果按高糖处理 | 收紧标签标准，明显高糖/高脂才显示标签，其余写入备注 |
| 营养单位不清晰 | 表格中只显示 kcal/g，没有统一说明按多少克计算 | 在表格上方统一提示“按每 100g 计算”，表头保持简洁 |
| Excel 部分数值格式不标准 | 有些单元格可能为空或包含混合文本 | 导入时解析第一个数字，无有效数字时按 0 处理 |
| 命令行导入不够直观 | 用户需要在页面中直接上传 Excel | 新增 `POST /api/foods/import` 接口和前端导入按钮 |
| 导入结果不便检查分类异常 | 只显示新增/更新/跳过数量，看不出分类是否异常偏多 | 导入结果新增 `categoryStats`，前端以分类小卡展示 |
| 茶叶蛋被识别成饮品 | 名称包含“茶”，旧规则先命中饮品关键词 | 调整关键词优先级，并新增茶叶蛋分类测试 |
| 牛肉面等复合食物容易误判 | 名称同时包含蛋白质和主食关键词 | 将主食判断放到蛋白质判断之前，并增加回归测试 |
| 导入结果弹窗不符合主题 | 使用 Element Plus 默认消息框，视觉偏普通后台系统 | 改为导入弹窗内部的主题化结果面板 |
| 删除确认弹窗不符合主题 | Element Plus 默认警告弹窗偏后台管理风格 | 为删除确认框增加专属 class 和主题化样式 |
| 6 月 1 日日志乱码 | 早期写入时发生编码错读后保存，文件本身已变成乱码 | 按用户确认后备份原文件，并根据可恢复内容重新整理 |
