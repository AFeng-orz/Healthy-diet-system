## 今日问题与解决

| 问题 | 原因 | 解决 |
|---|---|---|
| 饮食计划页面显得臃肿 | 每个食物项都重复展示推荐原因，且内容包含相同的饮食目标和健康画像说明 | 前端隐藏食物项长说明，只保留名称、克数、热量和三大营养素 |
| 食物卡片高度过高 | 文本说明撑开卡片，三餐区域纵向空间占用过大 | 压缩食物项间距，并用胶囊标签承载营养数据 |
| 饮食记录需要和目标值对比 | 单独记录食物只能知道吃了什么，缺少和健康档案目标的关系 | 每日记录响应中同时返回总摄入、目标值和完成率 |
| 用户可能误添加记录 | 饮食记录是高频操作，输入错误很常见 | Phase 8 第一版加入删除记录接口和前端删除操作 |
| 饮食记录页面局部风格偏默认后台 | Element Plus 的分段控件和消息确认框默认样式偏蓝色和后台系统风格 | 用页面 scoped 样式覆盖餐次选中态，并通过 customClass 主题化删除确认框 |
| 食物库和饮食记录删除弹窗风格不一致 | 两个页面分别维护删除确认样式，导致标题、提示区和按钮观感不同 | 抽出通用 `hd-danger-confirm` 样式，两个页面统一使用 |
| 健康报告缺少真实数据来源 | 报告页仍是占位页，无法体现饮食记录的追踪价值 | 基于 `t_diet_record` 聚合最近 7 天数据，生成今日达成率和热量趋势 |
| 首页内容仍停留在早期阶段说明 | 首页没有反映健康档案、饮食记录和报告等已实现能力 | 接入周健康报告接口，展示真实数据摘要和常用功能入口 |
| 页面横向空间偏窄 | 全局页面最大宽度仍是早期骨架阶段的 1180px，首页和列表页在桌面端显得不够舒展 | 将 `--hd-page-width` 调整为 1360px，顶部导航和页面主体统一变宽 |
| 页面切换瞬间出现方角 | 多数页面把 `v-loading` 挂在整体布局容器上，Element Plus 默认 loading mask 是矩形白色遮罩 | 增加全局 loading mask 主题样式，使用圆角、浅色半透明背景和绿色加载图标 |
| 常用入口图标显示异常 | 图片路径作为普通字符串使用，Vite 无法在组件中自动解析；健康报告图标文件名也与代码不一致 | 改为在 `Home.vue` 中显式 `import` 图片，并使用真实文件名 `dite.png` |
| 长页面浏览时顶部导航占用空间 | 顶部导航一直 sticky 显示，用户向下浏览记录、食物或报告时会遮挡一部分视野 | 在 `App.vue` 中根据滚动方向控制导航显示与隐藏 |
| 部分页面导航未及时隐藏 | 仅依赖 `window.scroll` 对部分页面的滚动方向感知不够稳定，且首页不应参与自动隐藏 | 增加 `wheel` 方向监听，并通过路由排除 `/home` |
| 根目录说明文档过期 | README 和 AGENTS 仍停留在 Phase 6，技术规范仍描述 Phase 1/2 骨架阶段 | 统一更新为 Phase 9 核心闭环完成、Phase 10 收尾的当前状态 |
| Logo 路径导致页面报错 | `:src="/logo/logo.png"` 会被 Vue 当成 JS 表达式解析，`/logo/logo.png` 又会被识别成非法正则 | 改为静态属性 `src="/logo/logo.png"` |
| 登录页专业词汇对用户不友好 | 登录页仍保留开发阶段和鉴权实现说明，更像技术展示而非用户入口 | 改为健康饮食规划说明，保留医疗建议边界，去掉技术标签和测试提示 |
